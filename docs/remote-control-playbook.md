# Remote Control Playbook — iPhone × Claude Code

_THINK YOU LAB / 最終更新: 2026-02-28_

---

## 1. 外出先 固定メニュー（iPhone から打つ3種）

### (1) 進捗確認
```
現在の進捗を教えて。
完了済み・進行中・次の予定タスクを箇条書きで。
```

### (2) 制約／品質条件追加
```
以下の制約を追加して、これ以降の実装に反映して：
- 触ってはいけない領域：[例: src/legacy/ は変更禁止]
- 互換維持：[例: v1 APIレスポンス形式を崩さない]
- テスト要求：[例: 変更箇所に必ずユニットテストを追加]
```

### (3) 次の一手（小さく刻む）
```
今の状態から次に進める最小ステップを1つだけ提案して。
先にテストを書く・ロールバック前提で進める、を優先して。
```

## 2. セッション命名ルール

| 要素 | 形式 | 例 |
|------|------|----|
| repo名 | そのまま | `my-app` |
| issue/目的 | 短い動詞句 | `fix-auth-bug` / `add-dark-mode` |
| 日付 | YYYYMMDD | `20260228` |

**完成形：** `my-app_fix-auth-bug_20260228`

- 同日に複数セッション：末尾に `-a` `-b` を付ける
- 長期タスク：日付は「開始日」固定（日をまたいでも変えない）

## 3. 中断・復帰の手順

### PC側（再開時に確認する場所）

- [ ] `git status` / `git diff` — 何が変更済みか確認
- [ ] `git log --oneline -10` — どこまで進んだか確認
- [ ] セッションの会話履歴を遡り「次のタスク」を確認
- [ ] `docs/` 配下のメモ・TODO を確認

### iPhone から再開する時

1. Claudeアプリ → Remote Control → PC側セッションを選択
2. セッションが見えない場合 → PC側で `claude` を再起動してセッションを新規作成
3. 最初に送るメッセージ：「現在の状況と次のタスクを教えて」

### セッションが消えていた場合

1. PC側で Claude Code を再起動
2. 対象repoのディレクトリで `claude` を起動
3. git log / git status で直近の状態を把握させる
4. 「〇〇の実装を再開して。直近の変更は git log で確認して」と指示

## 4. Remote Control 有効化チェックリスト

> **注意：Remote Control は Max プラン専用です。Pro プランでは利用不可。**
> Pro プランの場合はセクション6のフォールバック方針を参照。

- [ ] **Max プランであることを確認**（claude.ai でプラン確認）
- [ ] PC側の Claude Code を起動した状態で `/config` を実行
- [ ] 「Enable Remote Control for all sessions」を `true` に設定
- [ ] iPhoneの Claudeアプリを開く（同アカウントでログイン済みであること）
- [ ] アプリ内の Remote Control メニューからセッションが見えることを確認
- [ ] テスト用に簡単なメッセージを送り、PC側で受信されることを確認

## 5. Windows 直運用 — セッション継続ルール

### Windows Update / 再起動ルール
- 外出前に長時間タスクを走らせる場合は、Update を手動で延期（設定 → Windows Update → 更新を7日延期）
- 帰宅後・就寝前などPCを見ているタイミングで Update を実施
- 電源オプション：スリープを「なし」に設定しておく（長時間タスク中）

### 長時間タスクの分割戦略
- 1タスク = 30分以内で完結するサイズに分割
- 区切りポイント：テストが通る・ビルドが通る・コミットできる状態
- タスクの終わりに必ず「次のタスクは〇〇」とメモをコメント or ファイルに残す
- 長時間離席時は「ここまでコミットして止まって待って」と明示的に指示する

### セッション落ち・復旧チェックリスト
- [ ] PCを確認 → Claude Code プロセスが生きているか
- [ ] ターミナルを開き `cd [作業repo]` → `claude` で再起動
- [ ] `git status && git log --oneline -5` を実行して状態確認
- [ ] 「直前の作業状況を git log で確認して、続きを教えて」と送信
- [ ] 未コミットの変更がある場合：「変更内容を確認してコミットメッセージを提案して」

## 6. フォールバック方針（Remote Control 不安定時）

| 手段 | 判定 | 理由 |
|------|------|------|
| VPN / SSH で自宅PC到達 | **保留** | ルーターにポート開放不要なZerotier/Tailscaleは選択肢になりうる。ただし設定コスト・セキュリティ検討が必要。今は未設定なので実装しない。 |
| クラウドIDE（GitHub Codespaces等） | **保留** | 月額費用あり。ローカルリソース（Docker/ファイル）が使えない。緊急時の一時的な代替として検討可。 |
| iPhone Claude アプリでの直接会話 | **採用（暫定）** | Remote Controlなしでも新規セッションとして会話可能。コンテキストは引き継げないが、短文指示・方針確認には十分。 |
| PC操作の一時停止（帰宅後に再開） | **採用（原則）** | 外出中に致命的な判断が必要な局面は極力作らない設計が最善。タスク分割と事前整理で対応。 |
